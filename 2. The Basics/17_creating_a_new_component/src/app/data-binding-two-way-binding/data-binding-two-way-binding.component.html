Two-Way-Binding

Suppose we wnat to allow the user to add servers by clicking on the Button below:

{{ serverName }}

{{ ServerCreationStatus }}

But at this point, the button is just a static button is just a static button and it cannot do anything. So, we will add a new property in the TypeScript Code named allowNewServer as a boolean value. (Heading to TypeScript File to create the variable.)

In the constructor we will define a Timeout() method which will trigger after the specified period of time. In the method body, we will set the allowNewServer to be set to true and then use the disabled HTML attribute within Square Brackets ([]) as [disabled] which would indicate that we are using Property Binding to dynamically bind some property and change it in the DOM for the parsed model and then pass the property name used in the TypeScript class as: [disabled] = "!allowNewServer". This will exactly resolve to true or false based on the property needs.


Now, we have already created the Button. Say that we want to output the value of a Property (say we have a Property named ServerCreationStatus in the TypeScript File).

Now, to listen to events, we can create another method in the TypeScript File (named onCreateServer()). This method will be called by some event(s).

Now, we know that we want to listen to the click event generated by the Button (for which we have the onClick attribute in HTML as well). Now, in Angular, for Event Binding, we will use the syntax containing Parenthesis (()) and specify the name of the event within the Parenthesis as a click event and mention the code we want to execute within the Double Quotation marks which in our case is the onCreateServer() method.


Now, suppose before this button, we also have a label which consists of Server Name and we are allowing the user to provide the name of the server as input in the input field below it which would listen to the event. Now, in the TypeScript File, we want to update the value in the variable to the value which has been provided by the user in the input form. We can pass this value in the event listener in the form of $event which is a Reserved Variable Name for this event. It will represent the data emitted by the event. We will receive this event in the TypeScript File (any will be of type any for now and will be changed to Event when we will be assigning the data to some particular variable(s)).


We can bind to some other data in another way by combining Property Binding and Event Binding using [()] and along with this, we will use a special directive ngModel which we will set to some Property defined in the TypeScript code (for example, serverName) which will trigger on the input event and update the value of serverName in our component automatically and on the other hand, it will also update the value of the input element if we were to change the value of serverName to something else.
