Components

Components are important as we start with the AppComponent. It is a normal Angular Component and on the other hand it is also a special because it servers as our root component in the AppModule in the bootstrap array in the app.module.ts file which tells Angular that it is a special component and that we should bootstrap the entire application with that component being the root component. So, all the other components that we create will not be added to the index.html file and their selectors will not be added here as they will be added to the app.component.html file because it is the root component of the app with respect to the other parts.

Adding Component

Say, we want to output some information about a server as we are building a backend for a Server Application, so we can store it in a new folder which is the sub folder of the app folder because generally in an Angular CLI Project, all the app related content will be present within the app folder. Suppose the name of the sub-folder is Server because it holds the Server Component. It is a good practice to have the Folder Name equal to the Component Name and each component should have it's own folder.

Now, while creating the server component, the syntax which we can use is:

COMPONENT_NAME.component.ts

The file extension for the component is ts which refers to TypeScript in which we will code the component.

How to create a component

A Component is simply a TypeScript class so that Angular is able to instantiate it to create objects based on the blueprint that we will setup.

The naming convention for the TypeScript class is:

COMPONENT_NAMEComponent, where COMPONENT_NAME is the name of the component.

This is a normal class and that is the reason we need to tell Angular that this is a special type of class, a Component. We can do this by adding Decorators which are a special feature which allows us to enhance our classes using elements. Components are always added by using @Component before the class. And, since this is also unknown to the class, we will need to import the file which will inform about these components from the @angular/core. We have used @angular/core because Angular shifts between a lot of packages where it, basically, groups its functionality and core gives us access to some of the core functionalities.

Generating App Component using the Angular CLI

In the CLI, we have the generate command to generate some elements supported by Angular. We can create a component using the generate command with the following syntax:

ng generate component NAME_OF_COMPONENT OR ng g c NAME_OF_COMPONENT

For example, we want to create a component named servers, then we can generate that as:

ng generate component servers

Data Binding Components

Data Binding could be considered equivalient to Communication between the TypeScript Code, the Business Logic and the HTML Code because we might have some result in the TypeScript Code or we might have finished some calculation which we might want to display to the user but the only thing which the user is able to see is the template.

So, we need some sort of communicaiton between both pieces to be able to do something together. This is what Data Binding responsible for.

There are different ways of communication, for example, we want to out the data from our TypeScript Code to the HTML Code in the template, so we can either use String Interpolation or Property Binding for this.

Now, suppose the user clicks a button on the template, then we might want to trigger something on the TypeScript Code and we can achieve this with the help of Event Binding so that we can bind, for example, some click event to execute some code whenever it occurs.

There is also an additional Binding to combine the above methods known as Two-Way-Binding where we are able to react to events and output something at the same time.

So, basically, if we want to output something in the template (or print some text to it), then we can use String Interpolation whereas if we want to change some property of an HTML Element or a Directive or a Component, then we can use Property Binding. We CANNOT mix String Interpolation and Property Binding as it will break the code.
